# 업무 체크리스트 작성 스킬

## 사용 방법
"체크리스트 만들어줘" 또는 "/task-checklist [업무명]" 으로 호출

---

## 실행 순서

### 1단계 — 컨텍스트 수집 (먼저 질문)
시작 전 반드시 확인:
- 어떤 업무/마일스톤에 대한 체크리스트인가?
- 기간은? (스프린트 단위 / 런칭까지 / 특정 날짜)
- 담당자 구분이 필요한가? (PM / 개발 / 디자인)

### 2단계 — 체크리스트 유형

#### 개발 일정 관리용
```
## [기간] 개발 체크리스트

### 기획
- [ ] 요구사항 문서 작성
- [ ] 와이어프레임 확인
- [ ] 기능 우선순위 정의 (Must/Should/Nice)

### 개발
- [ ] 개발 환경 세팅
- [ ] API 설계 완료
- [ ] 핵심 기능 구현
- [ ] 코드 리뷰

### QA
- [ ] 기능 테스트
- [ ] 버그 수정
- [ ] 성능 테스트

### 완료 기준
- [ ] 모든 Must 기능 동작 확인
- [ ] 크리티컬 버그 0건
```

#### 런칭 준비용
```
## 런칭 체크리스트

### 앱스토어 등록
- [ ] 앱 아이콘 (1024x1024)
- [ ] 스크린샷 (6.5인치, 5.5인치)
- [ ] 앱 설명문 (한국어)
- [ ] 키워드 설정
- [ ] 연령 등급 설정
- [ ] 개인정보처리방침 URL 등록

### 기술
- [ ] 프로덕션 서버 점검
- [ ] 푸시 알림 테스트
- [ ] 결제 플로우 최종 확인
- [ ] 앱스토어 심사 제출

### 마케팅
- [ ] 런칭 공지 콘텐츠 준비
- [ ] SNS 채널 세팅
- [ ] 초기 유저 확보 채널 준비

### 법무/운영
- [ ] 이용약관 확인
- [ ] 개인정보처리방침 확인
- [ ] 고객 문의 채널 준비
```

#### 스프린트 계획용
```
## 스프린트 [N] 체크리스트 (기간: MM/DD ~ MM/DD)

### 목표
- 이번 스프린트 핵심 목표:

### 백로그 → 스프린트 이동
- [ ] 티켓 우선순위 정렬
- [ ] 개발 공수 산정
- [ ] 담당자 배정

### 완료 기준
- [ ] 목표 기능 구현 완료
- [ ] QA 통과
- [ ] Jira 티켓 업데이트
```

### 3단계 — 품질 기준
- 각 항목은 완료 여부를 명확히 판단할 수 있어야 함
- 모호한 항목 금지 → "개발 완료" (X) → "로그인 API 응답 200ms 이하 확인" (O)
- 담당자/기한이 있으면 항목 옆에 표시
- 우선순위 구분: 🔴 필수 / 🟡 권장 / 🟢 선택

---

## 출력 형식
요청한 업무 유형에 맞는 체크리스트를 마크다운으로 출력.
Jira 연동 시 티켓 생성 가능한 형식으로 작성.
