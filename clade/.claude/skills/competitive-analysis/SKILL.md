# 경쟁사 분석 스킬

## 사용 방법
"경쟁사 분석해줘" 또는 "/competitive-analysis [경쟁사명]" 으로 호출

---

## 실행 순서

### 1단계 — 컨텍스트 수집 (먼저 질문)
시작 전 반드시 확인:
- 분석할 경쟁사는? (특정 앱명)
- 집중할 기능/영역은? (온보딩, 수익화, UI 등)
- 비교 기준이 되는 우리 제품의 현재 상태는?

### 2단계 — 분석 구조

#### 기본 정보
- 앱명 / 출시일 / 주요 타겟
- 핵심 기능 요약
- 수익 모델 (무료/프리미엄/구독)

#### 무엇을 만들었나
- 핵심 기능과 차별점
- 주요 사용자층
- 경쟁사가 해결하는 문제

#### 강점 분석
- 가장 강력한 전략적 움직임
- 구체적 수치로 표현 (전환율, 리텐션, 가격 등)
- "좋은 UX" 같은 모호한 표현 금지 → "온보딩 3단계에서 이탈률 20% 감소" 처럼 구체적으로

#### 약점 및 기회
- 경쟁사의 공백 영역
- 유저 리뷰에서 반복되는 불만
- 우리가 공략할 수 있는 포인트

#### 우리 제품에 주는 시사점
- 즉시 적용 가능한 인사이트
- 피해야 할 실수
- 차별화 방향

### 3단계 — 품질 기준
- 모든 주장은 근거와 함께 (수치, 리뷰, 사례)
- 정보 공백은 `[필요: 데이터]` 로 표시, 추측 금지
- 디자인 미학보다 제품 의사결정에 집중

---

## 출력 형식
```
## [경쟁사명] 분석

### 기본 정보
### 핵심 기능
### 강점
### 약점/기회
### 스토익엔터테인먼트 AI 채팅 앱에 주는 시사점
```
