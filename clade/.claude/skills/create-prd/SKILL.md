# PRD 작성 스킬

## 사용 방법
"PRD 작성해줘" 또는 "/create-prd [기능명]" 으로 호출

---

## 실행 순서

### 1단계 — 컨텍스트 수집 (먼저 질문)
- 어떤 기능/이니셔티브에 대한 PRD인가?
- 주요 이해관계자는 누구인가?
- 개발 일정 제약이 있는가?
- 이미 결정된 사항이 있는가?

### 2단계 — 8섹션 PRD 구조

```
## 1. 요약 (Summary)
이 PRD가 다루는 내용을 2~3줄로 요약.

## 2. 담당자 (Contacts)
| 역할 | 이름 |
|------|------|
| PM | |
| 개발 리드 | |
| 디자인 | |

## 3. 배경 (Background)
- 왜 지금 이 기능이 필요한가?
- 어떤 문제를 해결하는가?
- 관련 데이터/유저 피드백

## 4. 목표 (Objective)
SMART 형식으로 작성:
- 목표: [측정 가능한 목표]
- 성공 지표: [KPI]
- 기간: [완료 기준일]

## 5. 타겟 사용자 (Market Segment)
- 주요 페르소나: 20~30대 여성
- 해결하는 문제: [구체적 문제]
- 사용 시나리오: [언제, 어떻게 사용하는가]

## 6. 가치 제안 (Value Proposition)
- 사용자가 얻는 혜택
- 경쟁사 대비 차별점

## 7. 솔루션 (Solution)
### 기능 목록
| 기능 | 우선순위 | 설명 |
|------|---------|------|
| | Must | |
| | Should | |
| | Nice | |

### 가정 사항
- [검증되지 않은 가정 목록]

## 8. 출시 계획 (Release)
- 출시 일정:
- 단계별 계획 (Phase 1 / 2):
- 리스크:
```

### 3단계 — 품질 기준
- 모든 목표는 수치로 표현
- 검증되지 않은 가정은 명시
- "사용자 친화적" 같은 모호한 표현 금지
- 공백 정보는 `[필요: ...]` 로 표시
