# 업무 현황 보고서 작성 스킬

## 사용 방법
"현황 보고서 써줘" 또는 "/status-update-writer [내용 요약]" 으로 호출

---

## 실행 순서

### 1단계 — 컨텍스트 수집 (먼저 질문)
- 보고 대상은? (대표/임원 / 팀장 / 개발팀 / 유관부서)
- 보고 주기는? (주간 / 월간 / 분기)
- 공유할 진행 상황 메모가 있는가?

### 2단계 — 대상별 보고서 구조

#### 임원/대표 보고 (핵심만, 200자 이내)
```
## [날짜] 주간 현황

**상태:** 🟢 정상 / 🟡 주의 / 🔴 블로킹

**TL;DR**
[가장 중요한 한 가지. 나쁜 소식은 여기에 먼저]

**이번 주 완료**
- [구체적 성과 — "온보딩 플로우 개발 완료" O / "개발 진행중" X]

**다음 주 계획**
- [항목] — 담당: [이름] / 기한: [날짜]

**리스크/블로킹**
- [리스크]: [대응 방안]

**결정 필요 사항** (있을 경우)
- [내용] — 추천: [의견] / 기한: [날짜]
```

#### 팀 내부 보고 (주간, 300자)
```
## [날짜] 팀 주간 업데이트

**스프린트 상태:** [N]% 완료

**완료된 작업**
- [티켓/기능명]: [결과]

**진행 중**
- [티켓/기능명]: [현재 상태] — 담당: [이름]

**블로킹 이슈**
- [이슈]: [필요한 지원]

**이번 주 집중 사항**
- [항목 1]
- [항목 2]
```

### 3단계 — 품질 기준
- TL;DR은 항상 첫 번째 — 나쁜 소식을 숨기지 말 것
- 활동이 아닌 **결과** 중심으로 작성
  - "마케팅 회의 진행" (X) → "런칭 채널 3개 확정" (O)
- 리스크는 반드시 대응 방안과 함께
- 수치로 표현 가능한 것은 반드시 수치로
- "대략 순조롭습니다" 같은 모호한 표현 금지
