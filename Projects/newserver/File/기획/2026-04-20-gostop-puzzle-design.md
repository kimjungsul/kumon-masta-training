# 고스톱 퍼즐 (웹 싱글 퍼즐 게임) 디자인 스펙

- **작성일**: 2026-04-20
- **상태**: 브레인스토밍 완료, 사용자 검토 대기
- **타임라인**: 6~8주 (권장 7주)
- **담당 PM**: 스토익엔터테인먼트 개발 PM

---

## 1. 제품 개요

### 한 줄 정의
고스톱 룰을 모르는 20~30대 캐주얼 유저가 5분 기본 튜토리얼 후 15개 정답형 퍼즐을 풀며 화투 카드 매칭 감각을 익히는 PC 웹 싱글 퍼즐 게임.

### 핵심 가치 제안
- 모바일 매치3에 식상한 유저에게 "생각하는 퍼즐" 경험
- 전통 화투의 비주얼 친숙함 + 체스 퍼즐의 "정답 찾기" 재미

### 핵심 차별점
- 기존 고스톱 게임은 대전 중심 + 중장년 UI → 본 제품은 솔로 퍼즐 + 모던 UI + 규칙 몰라도 진입 가능
- 화투 솔리테어류는 있지만 "최선의 수" 퍼즐 포지션은 드묾
- [필요: 경쟁사 분석 별도 진행]

### 타겟 유저
- 20~30대 캐주얼 퍼즐 유저
- 고스톱 규칙을 모름 (튜토리얼 필수)
- PC 데스크탑 브라우저 사용

## 2. 핵심 게임 루프

### 스테이지 진입 ~ 완료 흐름

1. **상황 제시**: 손패 7장 + 바닥 6장 + 더미 33장 (고정 초기 상태, 랜덤 없음)
2. **목표 제시**: 스테이지별 미션 (예: "광 3장 모으기", "고도리 달성", "7점 이상")
3. **턴 반복**: 손패 선택 → 바닥 매칭 → 획득 → 더미 뒤집기 → 매칭 → 특수 룰 처리
4. **완료**: 목표 달성 → 3별 평가 / 실패 → 재도전 무제한

### MVP 적용 룰 (전체 고스톱 룰)

| 룰 | 솔로 퍼즐 적용 |
|----|----------------|
| 월 매칭, 광/띠/피 수집 | 기본 |
| 쫑 (같은 월 2장 동시 획득) | 적용 |
| 쓸 (손패 같은 월 3장 즉시 획득) | 적용 |
| 따닥 (바닥 같은 월 2장 + 세 번째) | 적용 |
| 뻑 (바닥 같은 월 2장 → 처음엔 못 가져감) | 적용 |
| 흔들기 (손패 같은 월 3장 선언 → 2배 점수) | 선언 UI 필요 |
| 고도리 (홍단/청단/초단 족보 보너스) | 적용 |
| 똥광·쌍피 (카드 특성) | 적용 |
| **피박/광박** | **가상 상대 룰** — 스테이지 설정에 "가상 상대 피 개수" 파라미터 포함, 계산에만 사용 |

### 정답성 보장 방식
- 각 스테이지는 고정 초기 상태 + 고정 더미 순서 + 결정론적 룰 흐름
- 디자이너가 사전에 최소 N턴 풀이를 검증 → 3별 기준 설정
- 동일 입력 → 동일 정답 존재 보장

## 3. 튜토리얼 설계 (분할 방식)

### 구조

| 단계 | 내용 | 시점 |
|------|------|------|
| **기본 튜토리얼** (5분) | 기본 매칭 / 광·띠·피 / 기본 점수 | 첫 실행 시 |
| **기초 스테이지** (1~3) | 쫑·쓸·뻑 자연 체험 | 실전 시작 |
| **미니 레슨** (각 30초) | 따닥·흔들기 등 | 스테이지 4~6 진입 전 |
| **고급 미니 레슨** | 피박·광박·고도리 | 스테이지 7+ 진입 전 |

### KPI 정의
"튜토리얼 완주" = **기본 튜토리얼의 마지막 단계까지 모두 완료한 유저** (실제 소요 시간과 무관, 중간 이탈·스킵은 미완주로 간주)

기본 튜토리얼의 목표 길이는 평균 체감 5분 내외지만, KPI는 시간이 아닌 **완료 여부**로 측정.

## 4. 스테이지 구조

### 단일 스테이지 구성

| 요소 | 설명 |
|------|------|
| 고정 초기 상태 | 손패 7장 + 바닥 6장 + 더미 33장 순서 (디자이너 지정) |
| 목표(미션) | 1~2개 조건 (예: "광 3장" + "3턴 이내") |
| 가상 상대 설정 | 피박/광박 계산용 파라미터 |
| 허용 턴 수 | 3★/2★/1★ 기준 턴 수 |
| 해설(선택) | 클리어 후 풀이 팝업 |

### 15 스테이지 구성

| 구간 | 스테이지 | 학습 요소 | 난이도 |
|------|----------|-----------|--------|
| 기초 | 1~3 | 매칭·쫑·쓸·뻑 | ★ |
| 중급 | 4~6 | 따닥·흔들기 | ★★ |
| 고급 | 7~12 | 피박·광박·고도리 | ★★★ |
| 심화 | 13~15 | 전체 룰 조합, 3수 이상 읽기 | ★★★★ |

### 별점 기준 (3별 시스템)

최소 풀이 턴 수를 `N`이라 할 때:

- **3★**: `N`턴 이내 목표 달성
- **2★**: `N+1` ~ `N+2`턴 이내 목표 달성
- **1★**: `N+3`턴 이상 사용했지만 손패 소진 전에 목표 달성
- **실패(0★)**: 손패 소진 + 목표 미달

### 재도전 정책
- 무제한 재도전 (하트·리워드 광고 없음)
- 재도전 시 동일 초기 상태 (결정론적, 반복 학습 가능)

## 5. UI / 레이아웃

### 전체 화면 구성 (5개)
1. 스테이지 선택 — 15개 그리드 + 별점 표시
2. 기본 튜토리얼 (5분) — 카드 카테고리, 매칭, 점수
3. 게임 플레이 — 대시보드 3단 가로 레이아웃
4. 결과 — 별점 + 해설 + 재도전/다음
5. 설정(최소) — 사운드 ON/OFF, 튜토리얼 다시 보기

### 게임 플레이 화면 — 대시보드 3단 가로

```
┌──────────┬──────────────────────┬──────────┐
│ 미션     │                      │ 더미 +   │
│ 카드     │   바닥 카드 (6장)    │ 뒤집힌   │
│          │                      │ 카드     │
│ 점수/턴  │                      ├──────────┤
│ 카운터   │                      │          │
│          │   내 손패 (7장)      │ 가상상대 │
│ 내 획득  │                      │ (피 개수)│
│ 카드     │   액션 패널          │          │
│ (광·띠·피)│                     │          │
└──────────┴──────────────────────┴──────────┘
```

### 선택 근거
- 캐주얼 퍼즐 게임 표준 UX (좌측 정보·중앙 게임·우측 리소스)
- 미션·점수·획득이 항상 보임 → 인지 부하 감소 (규칙 모르는 타겟에 적합)
- PC 데스크탑 가로 공간 활용

## 6. 기술 방향

### 프론트엔드 (권장)
- **React + Vite**
- 스타일: CSS Modules 또는 TailwindCSS
- 게임 라이브러리(Phaser) 불필요

### 백엔드
- **없음** (게스트 + 로컬 저장 정책)
- 배포: Vercel / Netlify / Cloudflare Pages 중 택1 (정적 호스팅)

### 데이터 저장
- `localStorage` 스키마:
  ```
  {
    clearedStages: [{ id, stars, playedAt }],
    tutorialDone: boolean,
    lastPlayedStage: number
  }
  ```
- 백업 없음. 브라우저 데이터 삭제 시 초기화

### 화투 카드 자산
- **방식**: 기존 화투 아트를 참고한 캐주얼 리디자인 (자체 디자이너 제작)
- 월별 상징물·색 모티프만 차용, 오리지널 일러스트로 새로 그림
- 예상 기간: 2주 (Week 1~2, 병렬)
- 법적 리스크는 명시적으로 스킵 (사용자 지시)

### 계측
- 도구: GA4 (무료)
- 추적 이벤트:
  - `tutorial_started`, `tutorial_step_viewed`, `tutorial_completed`, `tutorial_skipped`
  - `stage_started`, `stage_cleared`, `stage_failed`
  - `mini_lesson_viewed`
  - `settings_opened`

### 성능 목표
- 초기 로딩 3초 이내 (카드 이미지 WebP)
- 턴 반응 100ms 이내

## 7. 성공 지표 (KPI)

### 1차 KPI

| 지표 | 정의 | 목표 |
|------|------|------|
| **튜토리얼 완주율** | 기본 튜토리얼 5분 끝까지 본 UV / 진입 UV | **≥ 70%** |

### 2차 지표 (방향성 참고)

| 지표 | 목표 |
|------|------|
| 첫 스테이지 클리어율 | ≥ 80% |
| 평균 플레이 스테이지 수 | ≥ 3 |
| Day 1 재방문율 | ≥ 25% |
| 스테이지 이탈 구간 | — (분석용) |
| 별 3개 달성률 | — (난이도 조정용) |

### 측정 제한사항
- 게스트·쿠키 기반 → UV 중복/손실 가능성
- 2차 지표는 방향성 참고용으로만 활용

## 9. 타임라인 & 리스크

### 7주 개발 일정

| 주차 | 주요 작업 | 산출물 |
|------|-----------|--------|
| Week 1 | 프로젝트 셋업 / 카드 데이터 모델 / 상태관리 기반 / 화투 디자인 착수 | 빈 페이지 + 카드 렌더링 |
| Week 2 | 룰 엔진 기본 / 화투 시안 | 매칭 플레이 가능 |
| Week 3 | 룰 엔진 고급 (쫑·쓸·뻑·따닥·흔들기·고도리·가상상대 피박/광박) / 화투 최종본 | 전체 룰 한 판 플레이 |
| Week 4 | 기본 튜토리얼 5분 + 미니 레슨 / 스테이지 1~5 | 튜토리얼 완결, 5스테이지 플레이 |
| Week 5 | 스테이지 6~15 / 별점 기준 / 결과 화면 | 15 스테이지 전체 플레이 |
| Week 6 | 스테이지 선택 UI / 설정 / 계측 연동 / 애니메이션 | 기능 완료 |
| Week 7 | 내부 QA / 외부 베타(10~20명) / 버그 수정 / 런칭 | **출시** |

### 주요 리스크 & 완화

| # | 리스크 | 완화책 |
|---|--------|--------|
| 1 | 화투 디자인 2주 소요 | Week 1부터 디자이너 병렬 착수, 임시 플레이스홀더로 로직 선행 |
| 2 | 튜토리얼 완주율 70% KPI | Week 4 종료 후 3~5명 사용성 테스트 → 조정 여지 확보 |
| 3 | 피박/광박 솔로 변형 UX 혼동 | 가상 상대 UI 단순화 (작은 카운터만), 미니 레슨 별도 |
| 4 | 정답성 검증 부담 | Week 3까지 풀이 검증 스크립트/시트 개발 |
| 5 | 게스트·로컬 저장 | 1차 KPI는 튜토리얼 완주율로 한정, 2차는 참고용 |
| 6 | 전체 룰 + 캐주얼 타겟 tension | 분할 튜토리얼 + 미니 레슨으로 인지 부하 분산 |

### 범위 조정 트리거
- Week 4 종료 시점에 Week 3 목표(전체 룰 플레이) 미완 → 스테이지 15→12 축소
- Week 5 종료 시점에 스테이지 10개 미만 완성 → Week 6 일정 재조정
